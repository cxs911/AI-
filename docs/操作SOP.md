# AI JD简历适配系统 - 操作SOP

## 一、系统启动

### 1.1 启动顺序

```
① 启动 Ollama 服务（后台自动运行）
② 运行 scripts/start.bat
③ 等待前端页面加载完成
④ 打开浏览器访问 http://localhost:3000
```

### 1.2 启动检查清单

- [ ] Ollama 服务正常运行（任务栏有 Ollama 图标）
- [ ] 所需模型已下载（`ollama list` 确认）
- [ ] Chrome 浏览器已安装
- [ ] Python 和 Node.js 已安装

## 二、使用流程

### 阶段一：准备个人素材

**目标**：将自己的简历/经历录入系统

**操作步骤**：

1. **上传简历**
   - 进入「个人素材库」
   - 点击「上传简历」
   - 选择 PDF/Word 格式的简历文件
   - 输入标题，选择分类
   - 点击「上传并解析」
   - 系统会自动调用Ollama提取结构化信息

2. **管理经历库**
   - 切换到「经历库」标签
   - 点击「添加经历」
   - 填写完整的经历信息（标题、类型、组织、时间、描述）
   - 特别注意「原始描述」要尽可能详细，这将影响AI优化效果
   - 保存后，点击「AI优化」按钮进行STAR量化转化

3. **创建标签分类**（可选）
   - 在素材库中管理标签
   - 方便后续按标签筛选

### 阶段二：JD智能解析

**目标**：添加目标岗位描述，让AI分析核心要求

**操作步骤**：

1. **添加JD**
   - 进入「JD智能解析」
   - 点击「添加JD」
   - 填写岗位名称、公司、城市等基础信息
   - **粘贴完整的岗位描述（JD全文）**
   - 点击「保存」

2. **AI解析JD**
   - 在JD列表中，点击对应JD的「解析」按钮
   - 等待AI完成能力拆解和权重打分（约10-30秒）
   - 解析完成后可点击「详情」查看

3. **查看解析结果**
   - 核心技能要求及权重
   - 关键词列表
   - 综合评分
   - 能力缺口（标注红色）

### 阶段三：生成定制简历

**目标**：针对具体岗位生成优化后的简历

**操作步骤**：

1. **生成简历**
   - 进入「简历管理」
   - 点击「生成简历」
   - 选择已解析的JD
   - 选择模板风格
   - 点击「开始生成」

2. **查看简历详情**
   - 进入简历详情页
   - 查看AI生成的定制化内容
   - 检查技能标签、经历排序是否合理

3. **生成招呼语**
   - 在简历详情页右侧
   - 选择风格（应届生/社招）
   - 点击「生成招呼语」
   - 可手动编辑修改

4. **导出PDF**
   - 在简历详情页点击「导出PDF」
   - 可选择预览后再导出
   - 导出的PDF为ATS标准格式

### 阶段四：Boss岗位检索与投递

**目标**：在Boss直聘上找到匹配岗位并投递

#### 4.1 浏览器登录

**⚠️ 首次使用必须手动扫码登录**

1. **启动浏览器**
   - 进入「投递管理」
   - 点击「启动浏览器」
   - 系统会自动打开Chrome窗口

2. **扫码登录**
   - 在打开的Chrome窗口中
   - 使用Boss直聘APP扫码登录
   - 系统会自动检测登录状态
   - **登录后会保存会话，下次无需重复登录**

#### 4.2 岗位搜索与筛选

1. **配置搜索条件**
   - 输入关键词（如：Python开发）
   - 选择城市
   - 设定薪资范围

2. **搜索岗位**
   - 点击「搜索岗位」
   - 系统自动在Boss上搜索并提取岗位信息
   - 结果会自动保存到投递列表

3. **筛选岗位**
   - 查看匹配度评分（0-100）
   - 注意过滤外包/培训岗位（自动标注）
   - 勾选需要投递的岗位

#### 4.3 投递操作

**两种模式**：

##### 模式一：手动单投（推荐，最安全）

1. 在岗位列表中点击某岗位的「投递」按钮
2. 系统自动执行投递操作
3. 每次投递后随机等待15-40秒

##### 模式二：批量投递

1. **必须先开启【人工审核开关】**
2. 勾选需要投递的岗位
3. 点击「批量投递」
4. 系统逐条执行投递，自动遵守风控规则

---

## 三、风控规则（系统自动执行）

| 规则 | 参数 | 说明 |
|------|------|------|
| 浏览器模式 | 可视化 | 禁止无头模式，显示真实浏览器 |
| 鼠标轨迹 | 随机偏移 | 每次操作前随机移动鼠标 |
| 页面停留 | 3-12秒 | 每次页面跳转后随机停留 |
| 每日上限 | 25个 | 达到上限自动停止 |
| 投递间隔 | 15-40秒 | 每次投递之间随机等待 |
| 工作时间 | 9:00-18:00 | 非工作时间不执行投递 |
| 工作日 | 周一至周五 | 周末不执行投递 |
| 人机验证 | 自动暂停 | 检测到验证码立即停止所有投递 |

## 五、常见问题

### Q1: Ollama连接失败
- 检查Ollama是否已启动
- 检查API地址是否正确（默认 http://localhost:11434）
- 检查防火墙是否阻止了连接

### Q2: 浏览器无法启动
- 检查Chrome是否已安装
- 尝试关闭杀毒软件
- 检查 `data/chrome_profile/` 目录权限

### Q3: 投递失败
- 检查是否在工作时间内
- 检查是否已达每日上限
- 检查是否触发了人机验证
- 查看后端日志 `logs/` 目录

### Q4: AI解析结果不准确
- 确保JD文本完整
- 尝试使用更大的模型（如 qwen2.5:14b）
- 检查Ollama服务是否负载过高
