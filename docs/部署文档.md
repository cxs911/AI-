# AI JD简历适配系统 - 部署文档

## 系统要求

| 组件 | 最低要求 | 推荐配置 |
|------|---------|---------|
| 操作系统 | Windows 10/11 64位 | Windows 11 |
| CPU | 4核 | 8核及以上 |
| 内存 | 8GB | 16GB |
| 磁盘 | 10GB可用 | 20GB SSD |
| Python | 3.10+ | 3.12 |
| Node.js | 18+ | 20 LTS |
| Chrome | 最新版 | 最新版 |
| Ollama | 已安装 | 已安装 + 7B+模型 |

## 前置安装

### 1. 安装 Python

1. 访问 https://www.python.org/downloads/ 下载 Python 3.10+
2. 安装时勾选 **"Add Python to PATH"**
3. 验证安装：
   ```bash
   python --version
   ```

### 2. 安装 Node.js

1. 访问 https://nodejs.org/ 下载 Node.js 18+
2. 安装时保持默认选项即可
3. 验证安装：
   ```bash
   node --version
   npm --version
   ```

### 3. 安装 Chrome 浏览器

1. 访问 https://www.google.com/chrome/ 下载安装
2. 确保版本为最新（Selenium会自动匹配驱动）

### 4. 安装 Ollama（可选，如需AI功能）

1. 访问 https://ollama.com/ 下载安装
2. 安装后打开终端，下载推荐模型：
   ```bash
   ollama pull qwen2.5:7b
   ```
3. 验证服务运行：
   ```bash
   curl http://localhost:11434/api/tags
   ```

## 部署步骤

### 方式一：一键启动（推荐）

双击运行 `scripts/start.bat`，脚本会自动完成以下操作：
1. 创建Python虚拟环境
2. 安装后端依赖
3. 安装前端依赖
4. 启动后端服务（端口 8000）
5. 启动前端服务（端口 3000）

### 方式二：手动部署

#### 后端部署

```bash
# 进入后端目录
cd backend

# 创建虚拟环境
python -m venv venv

# 激活虚拟环境
# Windows:
venv\Scripts\activate
# Linux/Mac:
# source venv/bin/activate

# 安装依赖
pip install -r requirements.txt -i https://mirrors.aliyun.com/pypi/simple/

# 启动服务
python -m uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

#### 前端部署

```bash
# 进入前端目录
cd frontend

# 安装依赖
npm install --registry=https://registry.npmmirror.com

# 开发模式启动
npm run dev

# 生产构建
npm run build
```

构建后的文件在 `frontend/dist/`，可用 Nginx 等部署。

## 访问系统

- API文档：http://localhost:8000/docs

## 配置说明

### 修改 Ollama 模型

在系统设置页面中修改：
- API地址：默认为 `http://localhost:11434`
- 模型名称：修改为已下载的模型，如 `qwen2.5:7b`、`llama3` 等

### 修改风控参数

在系统设置页面中调整：
- 每日投递上限（默认25）
- 投递间隔（15-40秒）
- 页面停留时间（3-12秒）
- 工作时间（默认 9:00-18:00）
- 工作日（默认周一至周五）

### 修改数据库路径

编辑 `backend/.env` 文件（如不存在则创建）：
```
DATABASE_URL=sqlite:///./data/jd_resume.db
OLLAMA_BASE_URL=http://localhost:11434
OLLAMA_MODEL=qwen2.5:7b
```

## 数据备份

所有数据存储在 `backend/data/` 目录下：
- `jd_resume.db` - 主数据库
- `uploads/` - 上传的原始文件
- `resumes/` - 解析后的简历文件
- `exports/` - 导出的PDF
- `chrome_profile/` - 浏览器登录会话

备份时只需复制 `backend/data/` 整个目录。
